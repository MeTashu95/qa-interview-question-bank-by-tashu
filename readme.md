Here are 15 important Manual Testing interview questions with short, easy-to-remember answers and examples for a beginner QA Manual Tester.

1. What is Software Testing?

Answer: Testing is the process of checking whether a software application works as expected and finding defects.

Example:
Check if the Login button allows valid users to log in.

2. What is a Test Case?

Answer: A test case is a set of steps, test data, and expected results used to verify a feature.

Example:

Step	Action	Expected Result
1	Enter valid username/password	User logs in successfully
3. What is a Defect/Bug?

Answer: A defect is a difference between expected and actual results.

Example:
Expected: Password should be hidden.
Actual: Password is visible.

4. What is SDLC?

Answer: Software Development Life Cycle is the process of developing software.

Phases:
Requirement → Design → Development → Testing → Deployment → Maintenance

5. What is STLC?

Answer: Software Testing Life Cycle is the process followed by testers.

Phases:
Requirement Analysis → Test Planning → Test Case Creation → Test Execution → Bug Reporting → Test Closure

6. What is Regression Testing?

Answer: Re-testing existing features after changes to ensure nothing is broken.

Example:
After adding a "Remember Me" option, verify Login still works.

7. What is Smoke Testing?

Answer: Basic testing to check whether the application is stable for further testing.

Example:
Check Login, Logout, and Dashboard loading.

8. What is Functional Testing?

Answer: Testing whether a feature works according to requirements.

Example:
Verify clicking "Add to Cart" adds the product.

9. What is Severity and Priority?
Severity	Priority
Impact of bug	Urgency to fix

Example:
Login not working = High Severity, High Priority.

10. What is Retesting?

Answer: Testing a bug again after the developer fixes it.

Example:
Verify password validation after the bug is fixed.

11. What is UAT?

Answer: User Acceptance Testing performed by clients/end users before release.

Example:
Client verifies the shopping process works correctly.

12. Difference Between Verification and Validation?

Verification

Are we building the product right?

Validation

Are we building the right product?

Example:
Requirement review = Verification
Testing application = Validation

13. What is a Test Scenario?

Answer: A high-level functionality to be tested.

Example:
"Verify Login Functionality"

Test cases:

Valid Login
Invalid Login
Empty Fields
14. What is Positive and Negative Testing?

Positive Testing

Valid data

Example:
Username: admin
Password: admin123

Negative Testing

Invalid data

Example:
Username: admin
Password: wrong123

15. How Do You Report a Bug?

Answer:
Include:

Bug ID
Summary
Steps to Reproduce
Expected Result
Actual Result
Severity
Screenshot

Example:

Bug ID: BUG-001

Summary: Login button not working

Steps:

Open Login Page
Enter valid credentials
Click Login

Expected: User logs in

Actual: Nothing happens

Interview Closing Question

"I have a good understanding of SDLC, STLC, test case writing, bug reporting, functional testing, regression testing, smoke testing, SQL, and Postman. I am detail-oriented, eager to learn, and focused on delivering quality software."

This set covers about 80–90% of the questions commonly asked in beginner/fresher Manual Tester interviews.
